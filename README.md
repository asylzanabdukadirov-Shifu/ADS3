Project Overview

В этом проекте я реализовал алгоритмы сортировки и поиска и сравнил их работу на массивах разного размера.
Целью было понять, как сложность алгоритма влияет на скорость выполнения.

Я использовал Bubble Sort как простой алгоритм, Merge Sort как более эффективный и Linear Search для поиска.

Algorithms

Bubble Sort проходит по массиву и меняет соседние элементы местами, если они стоят неправильно.
Он простой, но работает медленно на больших массивах.
Сложность O(n²).

Merge Sort делит массив на части, сортирует их отдельно и затем объединяет.
Он работает значительно быстрее на больших данных.
Сложность O(n log n).

Linear Search просто проверяет каждый элемент по очереди.
Сложность O(n).

Results

Я протестировал массивы размером 10, 100 и 1000 элементов.

На маленьких массивах все алгоритмы работают быстро.
При увеличении размера массива Bubble Sort становится заметно медленнее.
Merge Sort остаётся быстрым даже на больших данных.
Поиск выполняется быстро, но время увеличивается с ростом массива.

Здесь можно добавить реальные значения времени из программы.

Analysis

Merge Sort оказался быстрее, чем Bubble Sort, потому что его сложность ниже.

С увеличением размера массива разница между алгоритмами становится более заметной.

Отсортированный массив немного ускоряет Bubble Sort, но почти не влияет на Merge Sort.

Результаты соответствуют теоретической сложности алгоритмов.

Linear Search простой, но неэффективный на больших массивах.
Binary Search быстрее, но требует отсортированный массив.

Binary Search работает только с отсортированными данными, потому что он делит массив на части.

Reflection

В этом задании я понял, что простые алгоритмы не всегда эффективны.
На маленьких данных разница почти незаметна, но на больших она становится очень существенной.
